## <a name="authentication-library"></a><span data-ttu-id="bad85-107">Biblioteka uwierzytelniania</span><span class="sxs-lookup"><span data-stu-id="bad85-107">Authentication library</span></span>

* <span data-ttu-id="bad85-116">Użycie protokołu opartego na tokenach oferuje mniejszy obszar obszaru ataków, ponieważ tokeny nie są wysyłane we wszystkich żądaniach.</span><span class="sxs-lookup"><span data-stu-id="bad85-116">Using a token-based protocol offers a smaller attack surface area, as the tokens aren't sent in all requests.</span></span>
* <span data-ttu-id="bad85-117">Punkty końcowe serwera nie wymagają ochrony przed [fałszerstwem żądania między lokacjami (CSRF)](xref:security/anti-request-forgery) , ponieważ tokeny są wysyłane jawnie.</span><span class="sxs-lookup"><span data-stu-id="bad85-117">Server endpoints don't require protection against [Cross-Site Request Forgery (CSRF)](xref:security/anti-request-forgery) because the tokens are sent explicitly.</span></span> <span data-ttu-id="bad85-118">Dzięki temu można hostować Blazor aplikacje webassembly obok aplikacji MVC Razor lub Pages.</span><span class="sxs-lookup"><span data-stu-id="bad85-118">This allows you to host Blazor WebAssembly apps alongside MVC or Razor pages apps.</span></span>
* <span data-ttu-id="bad85-119">Tokeny mają węższe uprawnienia niż pliki cookie.</span><span class="sxs-lookup"><span data-stu-id="bad85-119">Tokens have narrower permissions than cookies.</span></span> <span data-ttu-id="bad85-120">Na przykład tokeny nie mogą być używane do zarządzania kontem użytkownika ani zmieniać hasła użytkownika, chyba że te funkcje są jawnie zaimplementowane.</span><span class="sxs-lookup"><span data-stu-id="bad85-120">For example, tokens can't be used to manage the user account or change a user's password unless such functionality is explicitly implemented.</span></span>
* <span data-ttu-id="bad85-121">Tokeny mają krótki okres istnienia, domyślnie jedną godzinę, która ogranicza okno ataku.</span><span class="sxs-lookup"><span data-stu-id="bad85-121">Tokens have a short lifetime, one hour by default, which limits the attack window.</span></span> <span data-ttu-id="bad85-122">Tokeny można również odwołać w dowolnym momencie.</span><span class="sxs-lookup"><span data-stu-id="bad85-122">Tokens can also be revoked at any time.</span></span>
* <span data-ttu-id="bad85-123">Samodzielna oferta JWTs gwarantuje klientowi i serwerowi proces uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="bad85-123">Self-contained JWTs offer guarantees to the client and server about the authentication process.</span></span> <span data-ttu-id="bad85-124">Na przykład klient ma środki do wykrycia i sprawdzenia, czy otrzymane tokeny są wiarygodne i są emitowane w ramach danego procesu uwierzytelniania.</span><span class="sxs-lookup"><span data-stu-id="bad85-124">For example, a client has the means to detect and validate that the tokens it receives are legitimate and were emitted as part of a given authentication process.</span></span> <span data-ttu-id="bad85-125">Jeśli osoba trzecia podejmie próbę przełączenia tokenu w trakcie procesu uwierzytelniania, klient może wykryć przełączony token i uniknąć korzystania z niego.</span><span class="sxs-lookup"><span data-stu-id="bad85-125">If a third party attempts to switch a token in the middle of the authentication process, the client can detect the switched token and avoid using it.</span></span>
* <span data-ttu-id="bad85-126">Tokeny z uwierzytelnianiem OAuth i OIDC nie polegają na tym, że agent użytkownika działa prawidłowo, aby upewnić się, że aplikacja jest bezpieczna.</span><span class="sxs-lookup"><span data-stu-id="bad85-126">Tokens with OAuth and OIDC don't rely on the user agent behaving correctly to ensure that the app is secure.</span></span>
* <span data-ttu-id="bad85-127">Protokoły oparte na tokenach, takie jak OAuth i OIDC, umożliwiają uwierzytelnianie i Autoryzowanie aplikacji hostowanych i autonomicznych z tym samym zestawem charakterystyk zabezpieczeń.</span><span class="sxs-lookup"><span data-stu-id="bad85-127">Token-based protocols, such as OAuth and OIDC, allow for authenticating and authorizing hosted and standalone apps with the same set of security characteristics.</span></span>
